# CI/CD Final Project

This context describes the MBA course deliverable: a public presentation of the project identity and its team. Its vocabulary keeps the submitted content distinct from the pipeline and deployment mechanisms that deliver it.

## Language

**Project title**:
The exact official title of the course deliverable: “Criação de um pipeline para Build e deploy de uma aplicação”.
_Avoid_: Application name, page title

**Team member**:
One person in the project team, identified by a full name and an RM number.
_Avoid_: User, account, contributor

**RM number**:
The institutional identification number paired with a team member in the submitted roster.
_Avoid_: ID, registration code

**Team roster**:
The complete, immutable set of the six agreed team members and their RM numbers for this deliverable.
_Avoid_: User list, member data

**Public roster page**:
The sole public representation of the deliverable, showing the project title and the complete team roster to a course evaluator.
_Avoid_: Dashboard, portal, home page

**Course evaluator**:
The person who verifies the submitted application, its delivery pipeline, and the required evidence.
_Avoid_: Customer, end user

**Submission evidence**:
The manually assembled material that demonstrates the repository, pipeline execution, and live application after delivery.
_Avoid_: Deployment artifact, application output
